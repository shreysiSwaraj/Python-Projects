

User Interface (UI) : UI is how human interacts with a system

ex. buttons, forms, menus, web pages, CLI prompts etc.

\- provide visual/command based interaction

In real life, ATM machine

screen

keypad

buttons





Application Interface (API) : API is how one software talks to another software.

ex. REST APIs, GraphQL, Webhooks, SDKs

\- provide data based interaction(JSON/XML)

In real life, An ATM machine talking to bank server

Check balance

Withdraw money





**Main HTTP request types:**



1. GET

fetch data from server

-read-only

-does not change anything on the server



2\. POST

send data to create something new

-used for creation

-data goes in request body



3\. PUT

updating an existing resource completely

-replaces old data with new data



4\. PATCH

update part of a resource

-partial update 

-more efficient than PUT



5\. DELETE

remove a resource
